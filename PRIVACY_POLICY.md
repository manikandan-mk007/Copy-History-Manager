# Privacy Policy for Copy History Manager

**Effective date:** April 23, 2026

Copy History Manager respects user privacy and is designed to give users control over their copied text history.

## 1. Information we collect

Copy History Manager may collect and store the following data:

- text that the user copies on webpages
- source page URL
- source page title
- timestamps related to copied items
- pin status and copy count
- account email address for signup/login
- extension settings such as tracking preferences, blocked domains, and sync preferences

## 2. How the information is used

The collected information is used only to provide the extension’s core functionality, including:

- saving copy history locally in the browser
- showing history inside the popup, history page, and quick paste modal
- filtering and organizing copied items
- syncing history to the user’s cloud account when the user chooses to log in
- restoring history across devices for the same user account

## 3. Local storage

By default, copied history is stored locally in the browser using extension storage.

This means:

- data remains on the user’s device unless cloud sync is enabled
- users can continue using the extension without creating an account

## 4. Cloud sync

If the user signs up or logs in, the extension may send copy history data to the configured backend server in order to:

- back up copy history
- restore history on another device
- keep a synchronized history for the same user account

Cloud sync is optional.

## 5. We do not sell personal data

Copy History Manager does not sell, rent, or trade user personal data to third parties.

## 6. Third-party services

If the backend is deployed using third-party hosting providers such as Render or a managed database provider, user data may be processed and stored through those infrastructure providers as part of the service operation.

## 7. Data security

Reasonable steps are taken to protect stored data, but no method of storage or internet transmission is 100% secure. Users should avoid copying highly sensitive information unless they fully trust their local machine and chosen backend deployment.

## 8. Sensitive data warning

Users are strongly advised not to use the extension to store highly sensitive data such as:

- passwords
- banking details
- government ID numbers
- medical records
- confidential corporate secrets

Although the extension can technically store copied text, it is not intended to be a secure secrets manager.

## 9. User control

Users can:

- disable tracking
- block selected domains
- delete individual history items
- clear all history
- choose whether to enable cloud sync

## 10. Account data

If users register for cloud sync, the backend may store:

- email address
- hashed password
- user-associated synced copy history
- user settings related to sync

Passwords should be stored only in hashed form and never in plain text.

## 11. Retention

Local data remains in browser storage until the user deletes it, clears extension data, or removes the extension.

Cloud-synced data may remain on the backend until deleted by the service owner or through a future user deletion feature.

## 12. Children’s privacy

Copy History Manager is not specifically directed to children under 13.

## 13. Changes to this policy

This Privacy Policy may be updated in the future. The updated version should include a new effective date.
